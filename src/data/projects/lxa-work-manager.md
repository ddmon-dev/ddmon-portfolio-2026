## 개요

Work Manager는 제 프리랜서 업무의 고객, 견적, 프로젝트, 일정, 수금, 파트너 정산 등을 정규화하여 관리하기 위해 제작한 개인용 업무 시스템입니다. 직접 제작해둔 admin-template을 기반으로, AI 에이전트를 활용해 구현했습니다.

이 프로젝트는 아직 익숙하지 않은 1인 외주 업무 흐름을 애플리케이션으로 정형화하고, 고객·견적·프로젝트·정산 데이터가 흩어지며 발생하는 관리 비용을 줄이기 위해 기획했습니다.

프로토타입은 Next.js와 로컬 Supabase 기반으로 빠르게 제작했고, 실사용 과정에서 React 클라이언트, Hono API 서버, Drizzle ORM, PostgreSQL 구조로 재구성했습니다.

## 작업범위

- 고객, 견적, 프로젝트, 일정, 정산 업무 흐름 정의
- admin-template 기반 관리 UI 구조 확장
- 워크플로 기반 데이터 모델 설계
- Docker PostgreSQL, LaunchAgent, 백업, 리빌드 스크립트 기반 로컬 운영환경 구성

## 주요 기능

- 고객 정보 관리
- 버전 기반 견적서 관리 / PDF 견적서 출력
- 프로젝트별 청구, 수금, 지출, 외주지급 장부
- 프로젝트 일정, 간트, 투두, 메모 위젯
- 파일 관리 드라이브 기능 (Global, 프로젝트, 고객별)

## 핵심 설계 결정

### Supabase/Next.js 기반에서 React/Hono 기반 구조로 변경

기존 admin-template을 활용해 Supabase/Next.js 기반의 프로토타입을 빠르게 제작한 뒤, 실사용 과정에서 React/Hono 기반 풀스택 구조로 변경했습니다.

개인 PC에서 상시 구동하는 업무 도구라는 조건에서는 Supabase 풀패키지를 계속 실행하는 것이 과하다고 판단했습니다.

실제로 필요한 것은 플랫폼 전체가 아니라 PostgreSQL 기반의 데이터 저장소였고, 조인과 트랜잭션이 필요한 업무 흐름은 명확한 API 서버와 ORM으로 다루는 편이 더 적합했습니다.

따라서 프론트엔드를 업무 데이터를 입력·조회·수정하는 가벼운 조작 레이어로 두고, 견적·프로젝트·정산 흐름은 API 서버와 데이터 모델에서 다루는 방향을 선택했습니다.

### 업무 흐름을 데이터 모델링으로 녹여내기

견적서는 고객 피드백에 따라 금액, 범위, 항목이 여러 번 수정될 수 있기 때문에 하나의 견적 번호 아래 여러 버전을 보존하는 구조로 설계했습니다. 계약 단계에서는 최종 버전을 지정하고, 해당 견적을 기준으로 프로젝트를 생성할 수 있도록 연결했습니다.

프로젝트 전환 이후에는 청구, 수금, 지출, 외주지급 흐름을 분리해 관리했습니다. 단순 금액 기록이 아니라 실제 외주 업무에서 필요한 계약 이력과 정산 흐름을 추적하는 데 초점을 맞췄습니다.

특히 견적서와 프로젝트 데이터를 어느 정도까지 연결할지가 핵심 과제였습니다. 견적에서 프로젝트가 생성되는 일반적인 흐름만 보면 최종 견적 버전을 강하게 참조하는 구조가 자연스럽지만, 실제 업무에서는 견적 없이 시작되는 프로젝트, 계약 이후의 범위 변경, 청구 대상 분리 같은 예외가 생길 수 있었습니다.

그래서 프로젝트는 견적 이력을 추적할 수 있도록 연결하되, 진행 상태와 정산 정보는 프로젝트 자체의 데이터로 독립적으로 관리하도록 분리했습니다. 프론트엔드에서 컴포넌트를 과하게 결합하면 예외 처리와 확장이 어려워지듯, 데이터 모델에서도 정합성과 유연성 사이의 경계를 조절하는 것이 중요했습니다.

### 개인 PC 대상 로컬 운영환경 구성

외부 사용자를 받는 서비스가 아니라 제 업무에 쓰는 내부 도구였기 때문에 클라우드 배포는 우선순위가 아니었습니다. 대신 개인 PC에서 항상 안정적으로 실행되고, 데이터와 파일이 로컬에 남는 구조가 더 적합했습니다.

운영 환경에서는 Docker PostgreSQL로 데이터 저장소를 고정하고, Hono 서버가 API와 빌드된 React 정적 파일을 함께 서빙하도록 구성했습니다. macOS LaunchAgent로 부팅 시 자동 실행되게 했고, 종료, 재빌드, 백업 스크립트로 운영 작업을 명령어화했습니다.

그 결과 외주 업무에 필요한 고객, 견적, 프로젝트, 정산 데이터를 로컬 환경에서 계속 누적해 사용할 수 있는 개인 업무 시스템으로 운영하고 있습니다.

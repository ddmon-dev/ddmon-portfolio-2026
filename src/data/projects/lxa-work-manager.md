## Overview

Work Manager는 프리랜스 업무의 고객, 견적, 프로젝트, 세무 흐름을 직접 쓰기 위해 만든 본인 IP 업무 도구입니다. 제가 유일한 사용자이기 때문에 요구사항을 누구보다 정확히 알고 있었고, 그 업무 흐름을 데이터 모델과 화면 구조로 옮기는 것이 프로젝트의 핵심이었습니다.

이 프로젝트에서 강조할 지점은 구현량보다 판단과 검증입니다. AI가 코드 작성 비용을 크게 낮춘 환경에서, 무엇을 만들지 정하고, 어떤 구조로 나눌지 판단하고, 나온 결과를 검토·디렉션하는 일이 실제 병목이라는 점을 보여주는 사례입니다.

## 작업범위

- 프리랜스 고객·견적·프로젝트·세무 관리 도구 기획
- 레거시 Next.js+Supabase 버전 제작 후 React+Hono+Drizzle로 전면 재작성
- FE 화면·UX, 도메인 모델, 데이터 구조, 운영 방식 설계
- 로컬 self-hosted 상시 구동과 일일 백업 환경 구성
- AI 드리븐 구현 결과 검토와 방향 디렉션
- 자연어 운영 스킬 레이어 실험

## 기획 의도

RGB Managers로 회사 내부 업무를 시스템화한 뒤, 제 프리랜스 업무도 고객 관리, 견적, 프로젝트 진행, 세금계산서, 원천징수, 외주 지급이 흩어진 상태라는 점이 보였습니다. 제 업무 흐름을 제가 가장 잘 알고 있었기 때문에, 기획자·사용자·개발자 역할을 모두 직접 맡아 도구로 옮겼습니다.

처음에는 Next.js와 Supabase로 빠르게 만들었지만, 한 명이 로컬에서 쓰는 도구에 여러 컨테이너의 Supabase 스택을 상시 구동하는 것이 과하다고 판단했습니다. 그래서 React SPA, Hono API, Drizzle ORM, PostgreSQL로 나누어 다시 만들었습니다.

## Key Features

- 고객 연락처, 메모, 환경 정보 CRM
- 견적 시리즈·버전 관리와 최종 버전 지정
- 부가세·원천징수 산식과 PDF 견적서 출력
- 프로젝트별 청구·수금·원가·외주지급 장부
- 매출·정산 통계, 일정·간트·투두·메모 위젯
- Claude Code 스킬을 통한 자연어 운영 실험

## 핵심 설계 결정

### 한 명이 쓰는 도구에 맞춰 Supabase 단일 앱을 4레이어로 분해

기능이 부족해서가 아니라 운영 맥락에 비해 스택이 무겁다고 판단해 전면 재작성했습니다.

- 사용자가 한 명이라 외부 웹 서비스보다 로컬 상시 구동이 적합.
- Supabase를 인증·실시간 없이 DB와 스토리지로만 써 BaaS의 장점을 충분히 쓰지 못함.
- 레거시의 다중 테이블 쓰기가 plpgsql RPC로 쌓일수록 비즈니스 로직이 DB 함수로 흩어질 위험.
- React SPA, Hono API, Drizzle 트랜잭션, PostgreSQL로 나눠 로직을 코드 레벨에 모으는 방향 선택.

### 견적 워크플로우를 테이블 불변식으로 모델링

가장 어려웠던 일은 코드 구현보다, 견적이 실제 업무에서 어떻게 바뀌고 프로젝트와 어떻게 연결되는지를 데이터 구조로 옮기는 일이었습니다.

- 견적을 시리즈, 버전, 그룹, 항목 4계층으로 정규화.
- 계약 상태와 최종 버전 지정이 어긋나지 않도록 DB 제약으로 강제.
- 프로젝트 돈 흐름을 청구, 수금, 원가, 외주지급 4개 장부로 분리.
- 부가세 별도 청구와 원천징수 3.3% 차감 같은 한국 프리랜스 업무 규칙을 모델에 반영.

### 금액 계산과 검증 규칙을 프론트·서버·PDF가 공유

화면에서 본 금액, 서버에 저장되는 금액, PDF에 출력되는 금액이 어긋나지 않도록 계산과 검증을 공유 패키지로 모았습니다.

- 같은 Zod 스키마가 프론트 폼 검증과 서버 입력 검증을 함께 구동.
- 부가세·원천징수 계산 커널을 폼, 상세, PDF, 서버가 공유.
- 서버는 클라이언트가 보낸 금액을 믿지 않고 다시 계산.
- 자동 타입 공유보다 명시적 패키지 경계를 택해 추적 가능한 구조 유지.

### AI 드리븐 구현에서는 의제 설정과 검증이 본인 역할이 됨

코드 대부분은 AI 도움으로 작성했습니다. 그래서 이 프로젝트에서 제 역할은 직접 손으로 모든 코드를 짜는 것이 아니라, 업무 문제를 정의하고 결과를 검토하며 방향을 교정하는 일이었습니다.

- 백엔드·모노레포 구현 세부는 AI에 맡기되 구조가 어떤 책임으로 나뉘는지 파악.
- 디자인 방향은 직접 잡고, AI 결과물을 텍스트 피드백으로 교정.
- 위험한 동작은 자연어 실행 전에 미리보기와 동의 게이트를 두는 방향으로 실험.
- 견적 산출 자동화는 가능성과 한계를 함께 확인한 진행 중 실험으로 정직하게 위치.

## 배포와 운영

웹서버로 공개할 필요가 없는 1인 도구라 제 머신에서 로컬 서비스로 운영합니다. 서버가 빌드된 프론트까지 함께 서빙하고, 개발/운영 환경을 분리했으며, 매일 DB 백업을 수행합니다. 지금도 일이 들어올 때 견적·프로젝트·세무 처리를 이 도구로 진행하고 있습니다.

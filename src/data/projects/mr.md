## Overview

(주)엠알은 2004년부터 의료용 전자기 자극기를 개발·제조해 온 의료기기 기업이고, 기존 홈페이지를 회원 인증·교육 콘텐츠까지 갖춘 다국어(국·영문) 사이트로 리뉴얼한 프로젝트입니다. 일반 기업 홈페이지와 달리 구매 의료진만 접근하는 회원 교육 영역이 핵심이고, 사용자 사이트와 운영 어드민 두 앱으로 구성돼 현재 mrev.co.kr로 운영 중입니다.

재직 중 마지막으로 끝까지 책임진 프로젝트로, 개발만이 아니라 PM으로서 기획·일정·디자인 기획까지 맡아 디자이너 1명과 2인 팀으로 완성했습니다. 게시판·어드민처럼 이전 프로젝트에서 검증된 구조는 이식해 빠르게 세우고, 그렇게 아낀 시간을 새로 설계해야 하는 회원 인증 시스템에 집중했습니다.

## 작업범위

- PM 겸 단독 개발: 기획·프로젝트 관리·디자인 기획(디렉션)까지 주도, 디자이너 1명과 2인 팀
- 사용자 사이트(home) 단독 신규 구축 + 운영 어드민(admin)은 기존 CMS 구조 이식
- home은 PC 시안만 제공받아 태블릿/모바일 반응형 직접 해석, 어드민 UI는 시안 없이 직접 설계
- 회원 가입·인증·교육 콘텐츠 접근 제어 전 구간 구현
- 국·영문 다국어 사용자 사이트와 운영 어드민

## 기획 의도

기존 PHP 기반 홈페이지를 현대적 스택으로 리뉴얼하면서, 단순 소개를 넘어 "구매 의료진에게 기기 교육을 제공하되 승인된 사용자만 접근"이라는 클라이언트 요구를 함께 충족해야 했습니다. 일반 기업 홈페이지엔 없는 회원 인증·콘텐츠 게이팅이 이 프로젝트의 핵심이었습니다.

요구사항 정의와 화면 설계, 일정까지 직접 잡았기 때문에 작업의 무게를 어디에 둘지도 직접 정했습니다. 게시판·어드민처럼 이전 프로젝트에서 검증된 구조는 이식해 구축 비용을 줄이고, 그 시간을 새로 설계해야 하는 회원 인증·교육 게이트에 투자한 것이 이 프로젝트의 작업 배분이었습니다.

## Key Features

- 회원 가입·로그인·관리자 승인 게이트·비밀번호 찾기
- 회원 전용 교육 영상 (최근 시청 이력)
- 국·영문 다국어 기업·제품 소개
- 콘텐츠 유형별 게시판(뉴스·갤러리·다운로드·영상·FAQ)·팝업
- 운영 모듈 14종을 다루는 어드민 CMS (CKEditor, 회원 승인, 알림메일 설정)

## 핵심 설계 결정

### 구매 의료진 전용 회원 인증·콘텐츠 게이트를 0부터 구축

구매 의료진만 교육 콘텐츠에 접근하도록, 일반 기업 홈페이지엔 없는 회원 인증 체계를 사용자 사이트에 직접 구축했습니다. 클라이언트 요구를 받아 가입부터 접근 제어까지 화면 뒤의 흐름을 끝까지 책임진 부분입니다.

- 가입 후 관리자가 승인해야 로그인되는 승인 게이트. 승인 시 어드민에서 다국어 안내메일 자동 발송.
- 보호 영역(교육 콘텐츠)은 서버에서 세션을 먼저 차단하고, 클라이언트가 세션 만료를 감시해 강제 로그아웃하며, 데이터 요청 시점에도 세션을 다시 확인하는 다층 방어.
- 표준 로그인 세션은 인증 라이브러리(NextAuth)에 맡기되, 라이브러리가 다루지 않는 미인증 플로우는 직접 설계(아래 트러블슈팅).

### 콘텐츠 성격별 다국어 저장 분기: 교육만 언어 컬럼 분리

게시판형 콘텐츠는 언어별 테이블로 분리하는 기존 방식을 그대로 쓰되, 교육 콘텐츠만 한 행에 언어별 컬럼을 두는 방식으로 다르게 설계했습니다. 같은 다국어 문제라도 콘텐츠 성격과 운영 방식이 다르면 저장 구조도 달라야 한다는 판단입니다.

- 교육 영상은 국·영문이 1:1로 동일함이 확정돼, 한 강의를 한 행으로 묶는 컬럼 분리(`title_ko`/`title_en`)가 자연스러움.
- 어드민에서 두 언어를 한 화면에 입력·관리하는 편이 운영 효율이 높다는, 데이터 모양이 아니라 운영자 편집 경험 기준의 선택.
- 치른 비용은 두 언어가 한 행에 묶여 한쪽만 등록할 수 없고 언어가 늘면 컬럼(스키마)을 바꿔야 한다는 것입니다. 1:1 확정·2개 언어 전제에서 받아들인 대가.

## 트러블슈팅 · 운영 한계 대응

### 비밀번호 찾기: 인증 라이브러리 밖 미인증 다단계 상태 관리

비밀번호 찾기는 사용자가 아직 로그인하지 않은 상태라 인증 라이브러리의 세션을 쓸 수 없는데도, 코드 요청 → 코드 검증 → 재설정 세 단계 동안 "누가 재설정 중인지"를 안전하게 이어가야 했습니다. 인증 라이브러리의 사각지대였고, 백엔드 흐름을 직접 설계해 풀었습니다.

- 신원 컨텍스트는 httpOnly·SameSite strict 쿠키에 10분 만료로 담아 단계 사이에 전달.
- 실제 인증 코드와 만료는 서버(DB)에 두고 매 단계 서버에서 검증, 재설정 완료 시 코드를 무효화해 일회성 보장.
- 메일 발송 실패 시 진행 세션을 정리하는 보상 처리로 어긋난 상태를 남기지 않음.

## 배포와 운영

현재 mrev.co.kr로 운영 중입니다. 이전까지 화면·기능 개발이 작업의 중심이었다면, 이 프로젝트에서는 기획·일정·디자인 기획까지 맡아 2인 팀을 이끌고 회원 인증을 갖춘 다국어 사이트와 운영 어드민을 함께 완성했습니다. 검증된 구조는 이식해 시간을 아끼고 새로 필요한 부분에 그 시간을 쓰는 작업 방식으로, 한 제품을 끝까지 책임지고 마감했습니다.

## Overview

(주)엠알 홈페이지는 의료기기 기업의 국·영문 기업/제품 사이트와 구매 의료진 전용 교육 콘텐츠를 함께 구축한 프로젝트입니다. 저는 PM 겸 단독 개발자로 기획, 일정 관리, 디자인 기획, 사용자 사이트와 어드민 개발을 맡아 디자이너 1명과 2인 팀으로 완성했습니다.

일반 기업 홈페이지와 다른 점은 회원 인증과 교육 콘텐츠 게이트였습니다. 단순 소개 사이트가 아니라, 기기를 구매한 병원·의료진만 교육 영상을 볼 수 있도록 가입, 관리자 승인, 로그인, 비밀번호 재설정, 보호 영역 접근 제어까지 직접 설계했습니다.

## 작업범위

- 사용자 사이트와 운영 어드민 개발
- PM, 기획, 일정 관리, 디자인 기획/디렉션
- home PC 시안 기반 태블릿·모바일 반응형 직접 해석
- 회원 가입, 관리자 승인, 로그인, 비밀번호 찾기 구현
- 교육 콘텐츠, 다국어 게시판, 팝업, 알림 메일 관리 구현
- 기존 어드민 골격 재사용과 MR 도메인 커스텀

## 기획 의도

기존 PHP 기반 홈페이지를 리뉴얼하면서, 구매 의료진에게 기기 사용법과 임상 교육 콘텐츠를 제공해야 했습니다. 아무나 볼 수 있는 소개 사이트와 승인된 고객만 접근하는 교육 영역을 한 서비스 안에서 함께 다뤄야 했기 때문에, 인증과 콘텐츠 게이트가 프로젝트의 중심이 됐습니다.

어드민은 기존에 만들었던 CMS 구조를 재사용해 구축 시간을 줄이고, 절약한 시간을 home의 회원 인증, 교육 콘텐츠, 다국어 반응형 화면에 집중했습니다. 마지막 재직 프로젝트였고, 개발뿐 아니라 프로젝트 진행 전반을 직접 끌고 간 작업입니다.

## Key Features

- 회원 가입, 관리자 승인, 로그인, 비밀번호 찾기
- 승인된 회원 전용 교육 영상과 최근 시청 이력
- 국·영문 기업·제품 소개와 다국어 게시판
- 뉴스, 갤러리, 다운로드, 영상, FAQ, 팝업 CMS
- 승인 안내 메일과 운영자 알림메일 설정

## 핵심 설계 결정

### 구매 의료진 전용 콘텐츠 게이트를 직접 설계

교육 영상은 기기를 구매한 병원·의료진에게만 제공되어야 했기 때문에, 일반 홈페이지에는 없는 가입 승인 흐름을 구현했습니다.

- 가입 후 관리자가 승인해야 로그인 가능한 승인 게이트.
- 승인 시 어드민에서 다국어 안내 메일 발송.
- 교육 콘텐츠 접근은 서버 레이아웃에서 먼저 차단하고, 클라이언트에서도 세션 만료 감지.
- 데이터 요청 시점에도 세션을 다시 확인해 화면 진입과 데이터 접근을 분리 방어.

### 인증 라이브러리 밖의 비밀번호 찾기 흐름을 별도 상태로 관리

비밀번호 찾기는 사용자가 로그인하기 전이라 NextAuth 세션을 쓸 수 없었습니다. 그러나 코드 요청, 검증, 재설정 3단계 동안 누가 재설정 중인지 안전하게 이어야 했습니다.

- 재설정 대상 이메일과 시작 시각은 httpOnly·SameSite strict 쿠키에 10분 만료로 저장.
- 인증 코드와 만료 시각은 DB에 저장하고 매 단계 서버에서 검증.
- 재설정 완료 후 코드를 null 처리해 일회성 보장.
- 메일 발송 실패 시 재설정 세션을 정리해 어긋난 상태를 남기지 않음.

### 다국어 저장 방식을 콘텐츠 운영 방식에 따라 분기

국·영문 콘텐츠를 한 방식으로 통일하지 않고, 콘텐츠 성격과 운영자 입력 경험에 맞춰 저장 구조를 나눴습니다.

- 정적 카피는 마크업 근처의 `content[lang]` 객체로 관리해 잦은 문구·개행 수정에 대응.
- 게시판형 콘텐츠는 언어별 테이블로 분리해 언어별 요구 변경이 서로 영향을 주지 않게 함.
- 교육 영상은 한/영이 1:1로 대응된다는 전제 아래 한 행에 언어별 컬럼을 둠.
- 데이터 정규화만이 아니라 운영자가 어떻게 입력·수정할지를 기준으로 구조 선택.

### 기존 CMS 골격은 재사용하고 MR 고유 흐름에 시간을 집중

어드민은 엔젤로보틱스에서 정리한 CMS 골격을 포크해 시작했습니다. 반복되는 관리 화면을 다시 짜는 대신, MR 고유의 회원 승인과 교육 콘텐츠 관리에 집중했습니다.

- 목록, 검색, 작성/수정 시트, CKEditor 업로드 등 공통 CMS 구조 재사용.
- 회원관리, 교육 콘텐츠, 연혁, 알림메일 설정 등 MR 도메인 모듈 추가.
- 어드민 구축 시간을 줄이고 home의 인증·교육 게이트 구현에 더 많은 시간을 배분.
- 재사용은 독립 자랑이 아니라, 프로젝트 완성 범위를 넓힌 수단으로 사용.

## 트러블슈팅 · 운영 한계 대응

### 미인증 다단계 상태를 세션 없이 이어야 했던 비밀번호 찾기

가장 까다로웠던 지점은 인증 라이브러리가 관리하지 않는 영역이었습니다. 로그인 전 사용자의 신원을 쿠키와 DB 코드 검증으로 나누어 보관하고, 각 단계마다 서버 검증을 거치게 해 인증 플로우 밖의 상태를 직접 설계했습니다.

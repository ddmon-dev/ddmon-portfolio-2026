## Overview

프리미엄 자동차 필름 브랜드 '파노라마 필름' 런칭과 함께 신규 구축한 **B2B 웹 서비스**입니다. 브랜드 사이트, 시공점 발주 시스템(OMS), 관리자 어드민 세 개의 앱으로 구성되며, 지역거점·시공점의 발주와 보증서 발급이라는 실제 업무 흐름을 처음부터 웹 화면으로 풀어냈습니다.

브랜드 사이트는 디자이너 1명과 협업해 제공받은 시안을 반응형으로 구현했고, OMS·어드민은 디자인 시안 없이 화면 설계부터 상태 관리, DB 설계, 배포, 고객사 커뮤니케이션까지 단독으로 담당했습니다.

## Highlights

- 시안 없는 OMS·어드민을 업무 요구에서 직접 설계했습니다 — 회원 유형(지역거점/시공점)에 따라 가격과 권한이 달라지는 발주 화면부터 PPF/틴팅 보증서 발급·출력, 매출 통계까지 OMS 23개·어드민 18개 화면의 플로우를 정의했습니다.
- 자체 폼·모달·훅 라이브러리(DD_LIB)와 도메인별 위젯 구조, 테이블만 바꿔 끼우는 제네릭 CRUD 패턴을 설계했고, 이 구조는 이후 Next.js 프로젝트들의 기반이 됐습니다.
- 발주 버튼 하나에 묶인 가격 검증·포인트 차감·알림톡 발송이 어느 단계에서 실패해도 화면과 데이터가 어긋나지 않도록, 서버 측 재검증과 보상 롤백까지 직접 챙겼습니다.
- 납품 후 현재까지 전국 100곳 이상의 지역거점·시공점이 사용 중이며, 유지보수 의뢰가 이어져 지속 수행하고 있습니다.

## Challenges

### 업무 요구를 화면 플로우로 — 시안 없는 OMS·어드민 설계

발주 시스템과 어드민에는 디자인 시안이 없었고, 고객사가 가진 것은 "지역거점과 시공점이 다르게 발주하고, 시공점이 고객에게 보증서를 발급한다"는 업무 요구뿐이었습니다. 이 요구를 회원 유형별 권한·가격, 발주·보증서 상태 흐름으로 분해한 뒤, 같은 발주 화면이 회원 유형에 따라 다른 가격과 메뉴로 동작하도록 화면 플로우를 설계했습니다.

상태 관리도 화면 설계의 연장이었습니다. 장바구니는 Zustand 스토어로 클라이언트 상태를 유지하면서 옵션 단위 수량 변경을 debounce로 묶어 서버와 동기화했고, 옵션이 모두 삭제되면 아이템까지 정리되는 정합성을 클라이언트에서 보장했습니다. 보증서는 A4 비율의 문서 레이아웃을 잡아 화면 그대로 PDF로 출력되게 했고, 발급된 보증서를 브랜드 사이트에서 고객이 직접 조회하는 흐름까지 연결했습니다.

권한 구조, 회원 유형별 다중 가격, 주문·보증서 번호 생성 규칙 같은 데이터 모델은 이 화면들이 거짓 없이 동작하도록 받치는 역할로 함께 설계했습니다.

### 첫 Next.js 프로젝트, 다음 프로젝트의 기반이 되도록

Next.js로 개발한 첫 프로젝트였습니다. 한 번 쓰고 버리는 구조로 만들 수도 있었지만, 이후에도 Next.js 기반 프로젝트가 계속 생기길 바랐기 때문에 프로젝트 구조에 가장 많은 고민을 들였습니다.

화면·상태·서버 로직을 도메인별로 묶는 위젯 구조를 잡고, 폼 컨트롤·모달 시스템·커스텀 훅처럼 프로젝트마다 반복되는 부분은 자체 라이브러리(DD_LIB)로 분리했습니다. 어드민은 리스트/폼 프로바이더를 제네릭하게 설계해 테이블만 바꿔 끼우면 새 관리 화면이 나오는 구조로 만들어, 18개 관리 화면을 같은 패턴으로 처리했습니다.

의도대로 이 구조는 한 프로젝트로 끝나지 않았습니다. 이후 진행한 Next.js 프로젝트들이 이 프로젝트의 구조와 라이브러리를 기반으로 착수됐습니다.

### 발주 플로우를 끝까지 — 무결성, 그리고 배포 전환

발주 버튼 하나에는 가격 검증, 주문 저장, 포인트 차감, 알림톡 발송이 묶여 있고 어느 단계에서든 실패할 수 있었습니다. 화면이 보여주는 결과와 실제 데이터가 어긋나지 않도록 금액은 Server Action에서 다시 조회해 재검증했고, 실패 시 주문 취소·보증서 삭제·포인트 복구를 묶은 보상 처리를 두었으며, 알림톡 실패는 발주 진행을 막지 않게 분리했습니다.

가장 큰 고비는 배포였습니다. 카카오 알림톡(비즈톡) API가 발신 서버의 고정 IP 등록을 요구해 Vercel에서는 연동이 불가능했고, 알림톡은 발주 플로우의 필수 요구사항이었습니다. 배포 환경 자체를 바꾸는 판단을 내려 가비아 클라우드 서버에 Nginx 기반 self-hosting 환경을 직접 구성했고, 이 전환으로 시스템을 요구사항 그대로 납품할 수 있었습니다. 인프라는 전문 영역이 아니라 필요해서 다룬 범위지만, 끝까지 완성하는 데 필요한 일이었습니다.

## Key Features

| 영역 | 기능 |
| --- | --- |
| 브랜드 사이트 | 반응형 브랜드 페이지(풀페이지 스크롤), 제품 소개, 대리점 찾기(지도), 보증서 조회·PDF 다운로드, 공지·FAQ·문의 |
| 발주 시스템(OMS) | 상품 발주·장바구니·발주 내역, PPF/틴팅 보증서 발급·출력, 시공점/상품/지역별 매출 통계, 공지사항 |
| 관리자 어드민 | 회원 승인·관리, 상품·카테고리 관리, 주문·배송 관리, 보증서 관리, 브랜드 사이트 콘텐츠 CMS, 엑셀 내보내기 |

## Result

납품 후 현재까지 큰 이슈 없이 운영 중이며, 전국 100곳 이상의 지역거점·시공점이 발주와 보증서 발급에 사용하고 있습니다. 납품 이후에도 기능 추가·유지보수 의뢰가 이어져 지금까지 직접 운영·유지보수를 맡고 있으며, 이 프로젝트에서 만든 구조와 라이브러리는 이후 Next.js 프로젝트들의 출발점이 됐습니다.

## Overview

솔라가드 한국본사의 프리미엄 브랜드 '파노라마 필름' 런칭에 맞춰 구축한 B2B 웹 서비스입니다. 브랜드 사이트, 시공점 전용 발주·보증서 발급 시스템(OMS), 운영 어드민 — 3개의 Next.js 앱으로 구성되며, 런칭 후 전국 100개 이상 시공점을 대상으로 운영 중이며, 현재까지 유지보수를 직접 수행하고 있습니다.

회사의 첫 Next.js 고객사 프로젝트로, 공통 구조와 모듈을 후속 프로젝트의 표준 기반으로 설계했습니다. 이 기반 위에서 7건의 프로젝트가 추가 수주·납품되었습니다.

## 작업범위

- 1인 단독 개발 — 3개 앱, 총 55개 화면, 4개월
- OMS·어드민 요구사항 정의 및 화면 설계
- 브랜드 사이트 개발 (디자인 시안 기반 반응형 구현)
- DB 스키마 설계, Supabase 기반 서버 로직
- 배포 환경 구성, 운영·유지보수
- 프로젝트 관리, 고객사 커뮤니케이션

## Key Features

- 상품 발주 · 장바구니 · 발주 관리, 주문번호 자동 생성
- 시공 필름 보증서 발급·조회, 보증서 PDF 다운로드·인쇄
- 발주·보증서 카카오 알림톡 발송 (비즈톡 API 연동)
- 매출 통계, 회원·상품·게시판·콘텐츠 관리 어드민

## Challenges

### 프로토타입 주도 설계로 발주 도메인 요구사항 확정

OMS와 어드민은 기획 문서·시안 없이 착수해, **동작하는 화면을 먼저 구현해 제안하고 고객사 피드백으로 수렴하는 프로토타입 주도 방식**으로 설계를 확정했습니다. 고객사 미팅으로 발주·보증서 발급 업무 플로우를 분석하고, 유사 커머스 서비스의 UX 패턴을 레퍼런스로 삼았습니다.

지역거점·시공점 2단계 권한 모델과 회원 유형별 차등 가격 정책도 이 과정에서 도출해 반영했습니다.

### 발주 플로우의 클라이언트 상태 설계와 데이터 정합성

상품 → 장바구니 → 주문 → 완료로 이어지는 멀티스텝 플로우에서 **클라이언트 상태와 서버 데이터의 정합성을 분리 설계**했습니다. 장바구니 수량 변경은 UI에 즉시 반영하되 DB 동기화는 300ms 디바운스로 묶어 응답성과 요청 빈도를 함께 확보했고, 주문 작성 상태는 Zustand persist로 영속화해 새로고침 시 입력 유실을 차단했습니다.

데이터 접근은 서버 전용 Supabase 클라이언트로 단일화하고(브라우저 클라이언트는 의도적으로 차단), 주문 제출 시 Server Action에서 회원 유형별 가격을 재검증한 뒤 중간 단계 실패 시 보상 처리로 주문을 롤백합니다. Supabase SDK가 멀티테이블 트랜잭션을 지원하지 않아 RPC(Postgres 함수)와 비교 검토했고, 당시 Postgres 숙련도 대비 유지보수 리스크를 근거로 Server Action 계층 처리를 선택했습니다. 비가역 작업인 알림톡 발송은 주문 확정 이후로 분리했습니다.

캐싱은 적용하지 않았습니다. 이 트래픽 규모에서는 가격 변경이 시공점 화면에 지연 반영되는 리스크가 캐싱의 이득을 상회한다고 판단해, revalidatePath 기반 온디맨드 갱신으로 일관시켰습니다.

### 제네릭 공통 구조로 17개 관리 화면 패턴화

어드민에서 반복되는 목록·검색·페이지네이션·작성/수정 골격을 추상화해 **17개 관리 화면을 하나의 제네릭 공통 구조로 구현**했습니다. 서버 컴포넌트가 데이터를 조회해 ListProvider / WriteProvider 컨텍스트로 하위 UI에 주입하는 계층 구조로, 신규 화면 추가와 납품 후 기능 확장 비용을 낮췄습니다.

필터·페이지네이션 상태는 URL 쿼리스트링을 단일 소스로 쓰는 useQueryString 훅으로 표준화하고, 폼 컨트롤·모달 시스템은 자체 라이브러리 DD_LIB로 분리했습니다. 이 자산들은 의도대로 이후 Next.js 프로젝트들의 공통 기반이 되었습니다.

### 고정 IP 제약에 따른 배포 아키텍처 전환

카카오 알림톡(비즈톡) API의 발신 서버 고정 IP 등록 요건으로 Vercel 배포를 유지할 수 없어 **배포 아키텍처를 재설계**했습니다. 알림톡 발송 서버만 분리하고 앱은 Vercel에 유지하는 안을 제안했으나, 고객사의 인프라 관리 일원화 결정에 따라 가비아 클라우드 셀프호스팅으로 전환했습니다. Nginx Proxy Manager 기반 리버스 프록시와 배포 환경을 직접 구성해 오픈했습니다.

## Overview

프리미엄 자동차 필름 브랜드 '파노라마 필름' 런칭에 맞춰 구축한 B2B 웹 서비스입니다. 브랜드 사이트, 전국 시공점이 쓰는 발주·보증서 시스템(OMS), 운영 어드민까지 3개의 Next.js 앱으로 구성됩니다. 백엔드 개발자가 없는 환경에서, 발주와 보증서 발급이라는 실제 업무 흐름을 화면 설계부터 데이터 구조까지 풀어냈습니다.

## Highlights

- 브랜드 사이트 15개 · 발주 시스템 23개 · 어드민 17개 — **3개 Next.js 앱, 총 55개 화면을 4개월에 단독 개발**해 오픈했습니다.
- 발주 시스템과 어드민 40개 화면은 기획서·디자인 시안 없이, 고객사 인터뷰로 업무 흐름을 파악하고 **동작하는 화면을 만들어 제안하며 직접 설계**했습니다.
- 새로고침에도 유지되는 주문 폼, 즉시 반응하는 장바구니 — 발주 플로우의 사용자 경험을 **서버 검증과 실패 복구까지 챙겨 완성**했습니다.
- 오픈 후 **전국 100곳 이상의 지역거점·시공점이 사용** 중이고, 퇴사 후에도 고객사의 직접 의뢰로 유지보수를 이어가고 있습니다.

## Challenges

### 기획서도 시안도 없는 발주 시스템, 화면 설계부터 시작

브랜드 사이트는 디자이너 시안이 있었지만, 발주 시스템과 어드민은 요구사항 문서와 레퍼런스 사이트만 전달받았습니다. 와이어프레임 없이 "시공점이 본사에 필름을 발주하고 보증서를 발급하는 업무"를 화면으로 옮겨야 했습니다.

문서로 기획을 확정한 뒤 개발하는 대신, 고객사 미팅으로 실제 업무 흐름을 파악하고 유사 커머스 서비스의 UX 패턴을 참고해 **동작하는 화면을 먼저 만들어 제안하고, 고객사 피드백으로 다듬는 방식**을 택했습니다. 발주 담당자가 실제 업무에 쓰는 도구라, 문서 합의보다 실물 화면에 대한 피드백이 빠르고 정확하다고 판단했습니다. 지역거점·시공점 2단계 권한 구조, 회원 유형별 가격 노출 같은 비즈니스 규칙이 이 화면들에 함께 담겼습니다.

### 발주 플로우 — 상태, 검증, 실패 복구까지 화면의 책임으로

상품 → 장바구니 → 주문 → 완료로 이어지는 발주 플로우는 화면 간 상태가 끊기지 않아야 했습니다. 장바구니는 수량 변경을 UI에 먼저 반영하고 DB 동기화는 300ms 디바운스로 묶어, 응답 지연이 사용감을 깎거나 쓰기 요청이 불필요하게 잦아지지 않게 했습니다. 주문 화면은 Zustand persist로 새로고침에도 작성 내용이 유지되게 했습니다. 발주 직전 단계에서 입력이 날아가는 것이 사용자에게 가장 치명적이라고 봤기 때문에, 요구사항에 없던 부분이지만 처음부터 설계에 넣었습니다.

화면이 "주문 완료"라고 말하면 데이터도 실제로 그래야 했습니다. Supabase는 SDK 단에서 멀티 테이블 트랜잭션을 지원하지 않아 RPC(Postgres 함수)가 정석이었지만, 당시 Postgres 숙련도로는 검증·유지보수 리스크가 더 크다고 판단해 **Server Action에서 가격 재검증과 실패 시 보상 롤백(주문 취소, 포인트 복구)을 직접 구현**했습니다. 새로고침으로 복원된 주문 입력도 제출 시점에 같은 검증을 다시 통과해야 합니다. 되돌릴 수 없는 외부 호출인 알림톡 발송은 주문 로직이 완전히 성공한 뒤에만 실행하고, 발송이 실패해도 주문에는 영향이 없도록 분리했습니다.

### 17개 관리 화면을 하나의 공통 구조로

어드민은 목록 + 검색 + 페이지네이션 + 작성/수정이라는 골격이 화면마다 반복됩니다. 매 화면을 따로 만드는 대신, 서버 컴포넌트가 데이터를 조회해 ListProvider / WriteProvider 컨텍스트로 하위 UI를 감싸는 **제네릭 공통 구조로 17개 관리 화면을 같은 패턴으로 처리**했습니다. 반복 구현을 줄이는 것과 함께, 납품 후 이어질 기능 추가·유지보수에 대비하려는 목적이 컸습니다.

어드민의 공통 구조와 함께, 폼 컨트롤·모달 시스템·커스텀 훅을 묶은 자체 라이브러리(DD_LIB)도 이 프로젝트에서 만들었습니다. 이 자산들은 의도대로 이후 Next.js 프로젝트들의 출발점이 됐습니다.

## Key Features

- 지역거점 / 시공점 2단계 회원 권한 구조와 유형별 가격 노출
- 상품 발주 · 장바구니 · 발주 관리, 주문번호 자동 발번
- PPF / 틴팅 보증서 발급, PDF 다운로드·인쇄, 브랜드 사이트 보증서 조회
- 발주·보증서 카카오 알림톡 자동 발송 (비즈톡 연동)
- 매출 통계, 상품·게시판·콘텐츠 관리 어드민

## Result

배포 환경까지 직접 결정해야 했습니다. 카카오 알림톡 API가 발신 서버의 고정 IP 등록을 요구해 Vercel을 그대로 쓸 수 없었고, 알림톡 서버만 고정 IP 서버로 분리하고 앱은 Vercel에 유지하는 안을 고객사에 제안했지만, 고객사가 인프라 관리 포인트를 하나로 두기를 원해 가비아 클라우드 일괄 self-hosting으로 전환해 오픈했습니다.

2024년 9월 납품 후 현재까지 특별한 장애 없이 운영 중이며, 전국 100곳 이상의 지역거점·시공점이 사용하고 있습니다. 퇴사 후에는 고객사가 직접 연락해 와 지금도 유지보수를 맡고 있습니다.

처음으로 Next.js를 고객사 프로젝트에 적용한 사례이기도 합니다. 이후 Next.js 기반 고객사 프로젝트 8건가량이 이어졌고, 이 프로젝트에서 만든 구조와 컴포넌트 자산이 그 출발점이 됐습니다.

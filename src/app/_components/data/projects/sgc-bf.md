## 개요

솔라가드 건축용 필름 브랜드의 보증서 관리 어드민과 조회 서비스입니다. 사전 개발해둔 Next.js 어드민 프레임워크를 기반으로 제작하였고, 공개 조회 사이트와 운영 어드민, API, 데이터 구조, 배포 환경까지 포함한 전체 제품 범위를 단독으로 설계·구현했습니다.

기존 파노라마 필름과 rgb Managers에서 경험한 보증서·문서 출력 흐름을 바탕으로, 2주 안에 운영 가능한 보증서 발급 시스템으로 런칭했습니다.

## 작업범위

- 보증서 조회 사이트 UI 디자인 및 개발
- 보증서 발급 관리 어드민 개발
- 보증서 PDF 시안 기반 @react-pdf/renderer 문서 구현
- 보증서 조회·PDF 발급 공개 API와 조회 흐름 구현
- 엑셀 대량 발급 검증 흐름 및 운영자 교정 UI 구현
- Supabase 기반 데이터 구조, 서버 액션, 배포 환경 구성

## 주요 기능

- 보증서 정보 기반 조회
- 보증서 PDF 미리보기·인쇄·다운로드
- 보증서 발급 및 발급 내역 관리
- 엑셀 기반 보증서 대량 발급
- 보증 제품 옵션·시공점·보증기간 관리

## 핵심 설계 결정

### 보증서 렌더링 스택 선정

기존 유사 프로젝트에서 경험한 html2pdf.js와 Puppeteer 기반 PDF 생성 방식의 한계를 바탕으로, 보증서처럼 고정 레이아웃에 가까운 문서에는 @react-pdf/renderer가 더 적합하다고 판단했습니다. Headless Chromium 의존성을 두지 않고 PDF 문서를 직접 구성하는 방식으로 렌더링 경로를 단순화했습니다.

- PDF 시안의 고정 레이아웃을 react-pdf primitive로 재현.
- 브라우저 기반 렌더링 없이 서버에서 PDF 버퍼를 직접 생성.
- 어드민 미리보기, 인쇄, 다운로드, 공개 사이트 PDF 발급 흐름에 같은 문서 정의를 사용.
- HTML/CSS 출력물과 PDF 출력물이 달라지는 문제를 줄임.

### 보증서 렌더링 경로의 단일화

어드민 미리보기, 공개 사이트 PDF 발급, 다운로드, 인쇄 경로가 서로 다른 문서가 되지 않도록 보증서 문서 정의를 하나로 유지했습니다.
기존 파노라마 필름 등의 프로젝트에서는 보증서 문서 렌더링을 두 앱에서 각각 수행했습니다. 본 프로젝트에서는 어드민 앱에서 보증서 문서 렌더링 API를 제공하고, 어드민과 조회 사이트가 같은 보증서 정의를 통해 미리보기, PDF 다운로드, 인쇄 흐름을 사용하도록 정리했습니다.

- 단일 react-pdf 보증서 컴포넌트를 클라이언트 미리보기와 서버 PDF 스트리밍에 함께 사용.
- 고객 정보 표시 규칙, 보증기한 계산, 문서 레이아웃 규칙을 한 문서 정의에 집중.
- 보증서 시안은 고객사가 제공했지만, 이를 PDF primitive와 서버/클라이언트 렌더 양쪽에서 재현.
- 환경 양쪽에서 렌더 가능한 컴포넌트로 유지해야 하는 비용을 수용.

### 보증서 조회 사이트의 책임 분리

공개 조회 사이트는 보증서 조회와 PDF 열람에 필요한 화면 흐름만 담당하도록 구성했습니다. 보증서 원본 데이터와 PDF 생성 로직은 어드민 앱에 두고, 조회 사이트는 API 호출을 통해 결과를 표시하는 구조로 분리했습니다.

- 조회 사이트 프로젝트에는 DB 접속 정보를 두지 않음.
- 보증서 조회와 PDF 발급은 어드민 API 라우트를 통해 수행.
- 공개 사이트는 조회 결과 표시, PDF 열람, 인쇄·다운로드 경험에 집중.

### 엑셀 대량 발급을 위한 검증 중심 워크플로우 설계

운영자가 여러 보증서를 한 번에 발급할 때 문제는 업로드 자체보다 잘못된 데이터가 조용히 발급되는 것입니다.

- 엑셀 행을 등록된 제품·VLT 옵션·시공점과 교차 검증.
- 검증 실패 행은 표 위에서 바로 수정하고 수정 즉시 재검증.
- 파일 내부 중복과 기존 발급 번호 중복을 각각 검사.
- 한 건이라도 틀리면 부분 성공 없이 다시 확인하게 해 증서 누락 추적 비용을 줄임.

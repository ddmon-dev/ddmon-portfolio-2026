## 개요

지원 서류를 잔뜩 검토하는 사람 입장에서 불편함과 피로를 줄이는 쪽으로 차별화를 노렸습니다. 수없이 많이 보셨을 화려한 연출은 걷어내고 스크롤과 읽기, 클릭만 남겼습니다. 그리고 눈에 거슬리지 않을 작은 디테일만 담았습니다.

구현은 AI(Claude Code)를 적극적으로 활용해 속도를 냈습니다. 대신 무엇을 만들지, 어디까지 다듬을지, 무엇을 버릴지는 직접 정했습니다. 이 판단이 가장 뚜렷하게 남은 곳이 Case Study 상세를 여는 방식의 시도와 결정이었습니다.

## 작업범위

- FSD와 App Router를 결합한 폴더 구조와 네이밍 컨벤션 설계
- 색·타이포그래피 토큰 기반 디자인 시스템 정의 (Tailwind v4)
- 프로젝트 갤러리와 상세 뷰 인터랙션 설계·구현
- Case Study 콘텐츠 작성 파이프라인 구성
- 한글 폰트 서브셋, 이미지 최적화 등 정적 자산 튜닝
- Vercel 배포

## 설계 포인트

### 프로젝트 갤러리 카드 UI 인터랙션

처음에는 갤러리 카드가 그대로 늘어나 다이얼로그로 펼쳐지는 framer-motion 레이아웃 애니메이션을 시도했으나 iOS 사파리에서 성능이 따라주지 않았고, 인터랙션 완성도도 기대에 못 미쳤습니다. 모바일 앱 환경에서는 자연스러운 패턴이지만, 웹에서 이 수준을 안정적으로 내기에는 아직 적합하지 않고, 개발 비용이 너무 과도하다고 판단하였습니다.

구현 비용 대비 인터랙션상의 임팩트를 줄 수 있을 것으로 기대되어 `vaul` 기반 하단 시트로 방향을 바꿨습니다. 확실하게 보장된 동작을 제공하면서, 데스크톱과 태블릿, 모바일 화면에서 각각 어색하지 않은 배경 스타일, 애니메이션을 분기하여 차별점을 주었습니다.

### 스크롤, 클릭, 읽기

카드를 눌러도 페이지를 옮기지 않고 그 자리에서 상세를 엽니다. 갤러리 카드를 누르면 인터셉팅 라우트가 페이지 이동을 가로채 하단 시트로 상세를 띄우고, 목록의 스크롤 위치와 맥락은 그대로 남습니다. 반면 주소를 직접 열거나 새로고침하면 인터셉트가 걸리지 않고 같은 내용이 정적 페이지로 로드되도록 하여 혹시 모를 새로고침으로 다시 카드를 열어야 하는 수고를 줄이고 싶었습니다.

### MD 파일 작성 DX 향상

케이스 스터디 글은 파일을 저장하면 새로고침 없이 화면에 바로 반영됩니다. 본문을 마크다운 파일로 두되, 파일 시스템에서 읽는 대신 모듈로 import하도록 구성해 개발 서버의 HMR이 걸리게 했습니다.

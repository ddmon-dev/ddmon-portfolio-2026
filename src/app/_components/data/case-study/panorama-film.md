## 개요

솔라가드 파노라마 필름 브랜드의 홈페이지, 시공점 발주 관리 서비스(OMS), 운영 어드민을 구축했습니다. 오픈 후 약 2년째 전국 78개 이상의 시공점이 사용 중이며, 유지보수는 지금도 직접 맡고 있습니다.

본사·지역거점·시공점 계층 간 상품 발주와 보증서·시공내역서 발급, 이 두 가지 핵심 요구사항을 중심으로 설계하고 구현했습니다. 이 프로젝트는 회사가 웹앱 제작으로 범위를 넓힌 첫 사례가 됐고, 이를 기반으로 같은 스택의 프로젝트를 6건 더 진행했습니다.

## 작업범위

- 브랜드 홈페이지 반응형 UI 구현
- 시공점 찾기, 게시판, 보증서 조회 기능 구현
- 시공점 발주, 장바구니, 주문서 작성, 보증서 발급 화면 구현
- 지역거점·시공점 등급에 따른 상품 노출, 주문 가능 액션, 관리 화면 흐름 분기
- 회원, 상품, 주문, 보증서, 게시판, 콘텐츠 관리 어드민 구현
- Supabase 기반 데이터 구조, 서버 액션, 배포 환경 구성

## 주요기능

- 본사(어드민)·지역거점·시공점 계층 기반 상품 발주, 장바구니, 주문 관리
- 보증서 발급, 조회, PDF 다운로드·인쇄
- 발주·보증서 카카오 알림톡 발송
- 매출 통계, 회원·상품·게시판·콘텐츠 관리 어드민
- 브랜드 사이트 내 시공점 찾기, 게시판, 보증서 조회

## 설계 포인트

### 본사·지역거점·시공점 계층별 주문 흐름 설계

한 서비스 안에 세 계층의 사용자가 공존하고, 회원 유형에 따라 메뉴 구성, 노출되는 샵과 상품, 주문 화면의 흐름이 갈라집니다. 특히 지역거점이 소속 시공점을 지정해 대신 발주하는 흐름은 일반 주문과 주문자·수령자·정산 주체가 달라, 주문 유형을 별도로 정의하고 화면과 데이터 양쪽에서 분기를 처리했습니다. 구현에 앞서 회원 유형별로 노출 기능과 가능한 액션을 기능 정의서로 정리하고, 이를 기준으로 개발 중 추가되는 요구사항을 판정했습니다.

### 주문 화면 상태 유지 방식

주문 화면은 상품 상세와 장바구니 두 경로에서 진입하는데, 구매 결정의 마지막 단계에서 새로고침 한 번에 주문 내용이 날아가면 안 된다고 판단했습니다. URL 파라미터, React 상태, sessionStorage를 검토한 끝에 Zustand persist를 선택해 수량 변경과 새로고침 복구를 같은 레이어에서 처리했고, 주문 완료 시 상태를 초기화했습니다.

### 보증서 / 시공내역서 렌더링 및 문서 출력

보증서·시공내역서는 화면 조회, PDF 다운로드, 인쇄를 모두 지원해야 했습니다. 문서를 HTML/CSS로 구현해 디자인 시안을 그대로 반영하고, 클라이언트에서 html2pdf.js로 PDF를 생성했습니다. 운영하면서 고해상도 출력과 인쇄 품질을 기대하기 어렵다는 점을 확인했고, 후속 프로젝트에서는 PDF 생성 방식을 바꿨습니다.

### 어드민 반복 구조의 설계

회원, 상품, 주문, 보증서, 게시판 관리는 도메인은 달라도 목록·검색·상세·수정이라는 화면 패턴이 같았습니다. 향후 프로젝트의 어드민에도 재사용할 것을 염두에 두고, 컴포넌트 구조와 비즈니스 로직 분리에 중점을 두고 설계했습니다.

### 알림톡 API 고정 IP 요구에 따른 배포 환경 결정

카카오 알림톡 API가 고정 IP를 요구해 Vercel 서버리스 환경에서는 쓸 수 없었습니다. 알림톡용 마이크로 서버 구축을 제안했지만, 고객사가 관리 포인트 증가를 우려해 최종적으로 가비아 클라우드 서버에 self-hosting으로 배포했습니다.

### 운영 중 회원·포인트 데이터 구조 개선

구축 당시 회원 정보와 포인트는 auth 테이블의 metadata에 두고 트리거로 별도 user_info 테이블에 복제하는 이중 저장 구조였습니다. 운영 중 포인트 이력 요구가 추가되면서 이 구조로는 대응이 어렵다고 판단해, 회원 데이터는 user_info 테이블 단일 관리로 정리하고 포인트 갱신은 DB 함수 하나로 모아 트랜잭션 안에서 이력과 함께 저장했습니다. 세션 조회 레이어는 기존과 같은 스키마를 반환하도록 유지해 화면·서버 액션 코드 수정을 최소화했습니다.
